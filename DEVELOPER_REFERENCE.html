Development Team Concerns — Status 7 items
✓ Fixed Relative paths broken in subdirectory pages
Pages inside services/ and locations/ were linking to index.html instead of ../index.html. All 48 subfolder pages have been corrected. Nav links, footer links, logo links, and CTA buttons all use correct relative paths.
href="index.html"  ← wrong from inside services/
href="../index.html"  ← correct
✓ Fixed Footer rendering as single column (UA stylesheet override)
Browser user-agent stylesheets were applying a white background and overriding grid layout on <nav> elements used as footer columns. All footers are now 100% inline-styled with no class dependencies — immune to any page-level CSS interference.
✓ Fixed Inconsistent nav and footer across pages
Each page had its own nav and footer with different structure, link order, button labels, and column counts. All 55 pages now share identical nav and footer HTML/CSS. WCAG SC 3.2.3 (Consistent Navigation) is met across the full site.
✓ Fixed Terms of Use references removed on legal advice
Lawyer advised not to use Terms of Use at this stage. All references removed from every page — nav, footer columns, footer-bottom legal links, and inline mentions. Confirmed absent across all 55 pages.
✓ Fixed Vendor names exposed in content
Financial Access page named Medicard, Stripe, and Juvonno explicitly including external links to their domains. All replaced with generic language: "our approved healthcare financing partner", "our digital payment plan provider", "our secure client portal". External vendor URLs replaced with internal contact.html.
✓ Fixed CTA section alignment inconsistency
Pre-Assessment Clinical Consultation block in green CTA sections had text-align:left hardcoded while the primary CTA was centred. Fixed to text-align:center on all affected pages. Block was also sitting outside the container div — corrected.
✓ Fixed CTA buttons — pre-launch popup added site-wide
All CTA buttons (Book Free Consultation, Patient Portal) now trigger a popup overlay instead of navigating to a dead-end page. The popup explains Toriven Health is launching late spring / summer 2026 and directs visitors to contact.html for more detail. Injected across all 59 pages. Contact page itself is exempt — buttons there behave normally. Footer links are also exempt.
Popup fires on: nav CTAs, hero CTAs, section CTAs, in-page booking buttons
Popup exempt: footer links, contact.html page itself
Accessible: Escape closes, overlay click closes, focus managed, aria-modal, aria-labelledby
⏳ Pending ™ trademark symbol — intentionally deferred
Trademark symbol removed from all pages pending trademark registration confirmation. When ready, a single script will add <span aria-hidden="true">™</span> to the logo on every page simultaneously — screen-reader silent, visually present. No content changes required.
